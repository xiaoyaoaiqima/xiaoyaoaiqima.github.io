寒假学一下做菜

炒菜

  1. 葱蒜都可以放，生姜考虑一下，如果要去腥味也可以放
  2. 用喷油，不要倒
  3. 肉类下去转一圈，把肉里面的油脂搞出来，再放没有油脂的东西，比如菜

烧菜

  1. 一般做比如炖牛肉，需要加一点山药，土豆，萝卜，这些先在锅里烧，差不多了再放牛肉，因为牛肉已经炖过了
  2. 肉类需要焯水，把血沫搞掉；新鲜的话，炒过的水还能用，比如炖牛肉，原汤炖；不新鲜就不能用
  3. 血沫怎么撇：把肉搞掉，再等烧开了水再撇

炖菜

  1. 高压锅一定要小心，可以放在水龙头下面冲一下
  2. 炖的时候，不用加很多料，只用加 20%的底味
  3. 25-30分钟
  4. 配料 香料提早煸，有些过油（八角）有些过水（香叶），到时候不用入锅了，直接搞菜里面
  5. 把牛肉，主料弄出来，作为菜帽，到时候辅料打底，菜帽放上面，高端

一、应用场景

  1. 社交媒体平台：

    • 在如今社交媒体盛行的时代，用户每天都会上传大量的照片。基于 Pix2Seq 的 image caption 可以自动为这些照片生成富有创意和准确性的描述，增强照片的表现力和故事性。例如，用户在旅行中拍摄了一张美丽的风景照片，该技术可以生成“夕阳余晖洒在古老的城堡上，如梦如幻的美景令人陶醉”这样的描述，让用户的分享更具吸引力。同时，这也有助于提高用户的互动性，因为生动的描述更容易引发朋友和粉丝的评论和点赞。
    • 对于企业和品牌来说，利用图像描述生成技术可以更好地推广产品和服务。例如，一家时尚品牌在社交媒体上发布新款服装的照片时，自动生成的描述可以突出服装的设计特点和风格，吸引潜在消费者的关注。

  2. 新闻媒体行业：

    • 新闻媒体经常需要快速处理大量的图片新闻。基于 Pix2Seq 的 image caption 可以为新闻照片自动生成简洁明了的标题和描述，提高新闻报道的效率和质量。例如，在报道一场自然灾害时，图片描述可以准确地传达灾害的严重程度和影响范围，让读者更直观地了解事件的情况。
    • 此外，对于一些视觉障碍人士来说，图像描述生成技术可以让他们通过听读描述来了解新闻图片的内容，提高新闻的可及性。

    • 在电子商务平台上，商品图片是吸引消费者的重要因素。基于 Pix2Seq 的 image caption 可以为商品图片生成详细的描述，包括产品的特点、功能、用途等，帮助消费者更好地了解商品。例如，对于一款电子产品，描述可以包括“高清屏幕，强大的处理器，长续航电池，满足你的工作和娱乐需求”，这样的描述可以提高消费者的购买意愿。
    • 同时，图像描述生成技术还可以用于商品搜索和推荐。通过分析图片的描述，电子商务平台可以更准确地为用户推荐相关的商品，提高用户的购物体验。

  4. 教育领域：

    • 在教育领域，图像描述生成技术可以为教学资源提供更多的辅助信息。例如，教师在讲解历史、地理、生物等学科时，可以使用带有描述的图片来帮助学生更好地理解知识。对于视觉障碍学生来说，图像描述生成技术可以让他们通过听读描述来了解图片的内容，提高学习的参与度和效果。

多臂老虎机

多臂老虎机（Multi-Armed Bandit，MAB）是一个经典的决策问题模型，在机器学习、强化学习和推荐系统等领域有广泛应用。

一、问题描述

想象有一个拥有 N 个臂的老虎机，每个臂对应一种不同的商品或选项。每次拉动一个臂，会产生一个随机的奖励，代表向用户推荐相应商品后用户可能带来的价值（比如购买商品带来的收益）。目标是在有限的时间内，通过合理地选择拉动哪个臂，最大化累计奖励，也就是最大化用户价值以便于用户购买商品。

二、探索与利用（EE 问题）

  1. 探索（Exploration）：

    • 这意味着推荐新的物品给用户，目的是开发用户潜在的兴趣点。因为如果只推荐已知表现好的商品，可能会错过一些实际上对用户价值更高但尚未被充分了解的商品。例如，向一个一直购买科技产品的用户推荐一本新的畅销小说，可能会发现用户其实也对文学作品有兴趣。
    • 通过探索，可以拓宽对用户兴趣的了解范围，为未来的推荐提供更多可能性。

  2. 利用（Exploitation）：

    • 利用当前已获得的信息来最大化收益。即根据过去的经验，选择那些被证明能够产生较高奖励的臂（商品）进行推荐。例如，如果过去发现某款电子产品在用户中很受欢迎，那么就倾向于继续推荐这款产品以获取稳定的收益。
    • 然而，过度利用可能会导致错过新的更好的机会。

三、Epsilon-greedy 方法

Epsilon-greedy 是解决多臂老虎机问题的一种常见策略。

  • 设定一个参数 ϵ（epsilon），通常是一个介于 0 和 1 之间的小数。在每次决策时，以 ϵ 的概率进行探索，即随机选择一个臂；以 1 - ϵ 的概率进行利用，选择当前平均奖励最高的臂。
  • 例如，如果 ϵ = 0.1，那么在每次决策时有 10%的概率随机选择一个商品推荐给用户，进行探索；有 90%的概率选择过去表现最好的商品推荐，进行利用。
  • 这种方法在开始时会进行较多的探索，随着时间推移，逐渐更多地进行利用，以平衡探索新机会和利用已知好选项之间的关系。

一、IoC（Inversion of Control，控制反转）

  1. 概念解释：

    • 在传统的编程模式中，对象的创建和依赖关系的管理通常由对象自己负责。例如，一个对象 A 如果需要依赖对象 B 来完成某些功能，那么对象 A 会直接实例化对象 B。而在 IoC 模式下，这种控制权发生了反转，对象的创建和依赖关系的注入由外部的容器（如 Spring 容器）来负责。
    • 简单来说，就是将组件的创建和配置的控制权从组件自身转移到了外部的容器。

  2. 作用和优势：

    • 解耦：通过 IoC，组件之间的依赖关系变得更加松散。各个组件不需要直接了解其他组件的具体实现，只需要依赖抽象接口。这使得系统更容易维护和扩展，因为当一个组件发生变化时，不会对其他组件产生直接影响。
    • 提高可测试性：在测试环境中，可以方便地为被测试对象注入模拟的依赖对象，而不需要实际创建复杂的真实对象，从而提高测试的效率和可靠性。
    • 增强代码的可复用性：由于组件的创建和配置由容器管理，不同的组件可以在不同的场景下复用，只需要在容器中进行适当的配置即可。

二、AOC（Aspect Oriented Computing，面向切面编程）

  1. 概念解释：

    • AOC 是一种编程范式，它允许开发者将横切关注点（cross-cutting concerns）从业务逻辑中分离出来。横切关注点是指那些在多个模块中都存在的功能，如日志记录、安全检查、事务管理等。
    • 在传统的编程方式中，这些横切关注点通常会分散在各个业务模块中，导致代码的重复和混乱。而 AOP 通过将这些横切关注点封装成切面（aspect），在不修改原有业务逻辑的情况下，将这些功能动态地织入到目标对象中。

  2. 作用和优势：

    • 代码整洁性：将横切关注点从业务逻辑中分离出来，使得业务代码更加清晰和专注于核心业务功能，提高了代码的可读性和可维护性。
    • 避免代码重复：对于多个模块都需要的功能，如日志记录和安全检查，不需要在每个模块中重复编写代码，只需要在切面中定义一次，然后在需要的地方进行织入即可。
    • 提高可扩展性：可以方便地添加新的横切关注点，或者修改现有的横切关注点，而不需要对业务逻辑进行大规模的修改。

总的来说，IoC 和 AOC 都是为了提高软件系统的可维护性、可扩展性和可测试性而出现的编程理念和技术，在现代软件开发中得到了广泛的应用。
